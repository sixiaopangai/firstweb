Java Web 开发工作流 & 文件夹规划

构建高效、整洁、易于 AI 协作的开发环境

1. 物理位置与分层结构

统一存放路径：D:\Workspace\Java-Web\

D:\Workspace\Java-Web\
├── 01_Learning/           # 练习项目、Demo、实验课作业
├── 02_Active/             # 正在进行的核心开发项目
├── 03_Templates/          # 存放脚本、README模板、脚手架
│   ├── clean_project.py   # Python 清理核心
│   └── 一键清理.bat        # Windows 快捷工具
└── 04_Archive/            # 已清理垃圾后的“瘦身”归档项目

2. 单个项目内部规范

推荐采用前后端分离结构，方便多个 AI IDE 协同工作：

文件夹 职责 推荐 IDE / 工具
backend/ Java 源码、Maven 配置 IntelliJ IDEA
frontend/ Vue/React 源码、Node 依赖 Cursor / Trae
sql/ 数据库初始化及变更脚本 Navicat / Datagrip
docs/ 项目设计文档、接口定义 Typora / Obsidian

3. 自动化清理与归档流

归档秘籍： 在将项目移入 04_Archive 前，务必执行清理脚本。将clean_project.py和一键清理.bat放入想要清理的文件夹下，执行一键清理.bat。执行完即可把项目文件夹放入04_Archive

运行脚本将自动删除以下冗余内容，节省 SSD 空间：

  • target/ (Java 编译产物)
  • node_modules/ (前端巨量依赖)
  • .idea/ & .vscode/ (本地 IDE 索引)
  • dist/ & build/ (构建打包结果)

4. 恢复运行命令

当需要从归档中重启项目时，在对应目录下执行：

# 后端恢复 (backend/)
mvn clean install

# 前端恢复 (frontend/)
npm install
npm run dev
